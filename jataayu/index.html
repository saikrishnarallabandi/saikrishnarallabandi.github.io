// the problem

AI agents are under attack —
and they also leak.

A poisoned GitHub issue hijacks your coding agent. An invisible instruction on a web page redirects your browsing agent. A crafted email makes your assistant exfiltrate data. Most defenses only watch what comes in. But there's a second threat nobody talks about: your agent holds private context — files, messages, family details, finances — and can leak it the moment it replies in a group chat or comments on an issue. No attacker required. Jataayu guards both directions.

inbound · attack

Catch the injection

Prompt & command injection, jailbreaks, social-engineering, homoglyphs and encoded payloads — flagged before your agent ever acts on the text.

outbound · privacy

Stop the leak

PII, secrets, minors' info, health and financial details, protected names — detected and auto-redacted before the reply ever leaves.

surface-aware

Context matters

A shell command is suspicious in a GitHub issue, expected in a coding task. Every surface gets its own trust level and strictness.

open · honest

No black box

Reports what it found and why, with a risk score. MIT-licensed library, CLI and MCP gateway hook. 344 tests, reproducible benchmarks.

// how it works · layered defense

The boundary isn't the attack string.
It's what the action does.

Matching attack text is the weakest tier — an adaptive attacker just rewrites the string. Jataayu keeps a fast regex layer as a cheap pre-filter, then moves the real boundary to action-level authorization, following the 2026 agent-security literature (CaMeL, FIDES, SecAlign). Four layers, defense-in-depth.

Layer 0 · normalize

See through evasion

Multi-view normalization — NFKC, homoglyph fold, zero-width strip, de-space, de-leet — plus recursive base64/hex/url decode. An attacker must evade every view at once.

Layer 1 · effect boundary

Authorize the effect

Deterministic PREVIEW→COMMIT. Provenance-typed values × effect severity × capability policy decide ALLOW / DENY / NEEDS-APPROVAL. No LLM in the loop.the security floor

Layer 2 · learned

A model that ignores injection

SecAlign-style SFT+DPO tuning of a structured-query model that obeys the instruction channel and ignores instructions buried in data. Defense-in-depth, never the floor.

fast path · regex

Microsecond pre-filter

100+ patterns across injection, command-exec, social engineering, encoding. Sub-millisecond; escalates ambiguous cases to an LLM slow path.

// use it · one import, two guards

Drop it into your agent

A library first — call it before your agent acts on external content, and again before it sends anything to a shared surface. Also ships as a CLI and an MCP gateway hook.

agent.py 
from jataayu import jataayu_check_inbound, jataayu_check_outbound

# --- INBOUND: detect injection before your agent acts ---
res = jataayu_check_inbound(issue_body, surface="github-issue")
if res["status"] == "HIGH":
    raise SecurityError(res["findings"])   # blocked

# --- OUTBOUND: strip PII/secrets before you send ---
res = jataayu_check_outbound(reply, surface="discord-channel",
                            protected_names=["Veda"])
safe_text = res["redacted"] if res["status"] != "SAFE" else reply
